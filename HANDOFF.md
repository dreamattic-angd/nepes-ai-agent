## 성공한 접근법
- **NAA에서 먼저 검증**: 1인 프로젝트라 승인 절차 등을 생략하면서 체계만 잡을 수 있음
- **Loop 문서 형식**: 코드블록 안에 넣어서 붙여넣기하면 Loop에서 깔끔하게 표시됨. 단, `~` 문자는 취소선으로 변환되므로 사용 금지
- **ASCII 다이어그램 회피**: Loop에서 깨지므로 텍스트 설명으로 대체
- **ITSM 티켓 연동**: nGit 이슈 대신 사내 ITSM 티켓(#ITSM-XXXX)을 커밋 메시지에 포함
- **태그 소급 적용**: git tag -a {버전} {커밋해시} -m "{메시지}"로 과거 태그도 수정 가능

## 실패한 접근법
- **nGit 이슈 사용 계획**: 사용자가 ITSM만 사용하기로 결정. nGit 이슈는 등록하지 않음
- **/naa:cm-audit 첫 실행**: 커맨드 호출 후 "진행하겠습니다"만 출력하고 실제 점검이 안 되는 문제 발생. 에이전트 파일을 읽고 바로 실행하도록 해야 함
