⚙️ 원칙

Engineering Principles

Architecture Decision Records (DESIGN.md)에서 정의한 핵심 설계 원칙

1. Phase 기반 워크플로우

Lazy load로 토큰 최소화, 체크포인트 기반 재개, Phase별 독립 수정

2. Hook 기반 안전 장치

프롬프트 규칙 대신 Node.js PreToolUse Hook으로 우회 불가 차단 (exit code 2)

3. JSONL 로그 포맷

Append-only, 한 줄 손상 격리, 스트림 처리 가능

4. 파일 기반 타이머

Phase별 별도 프로세스 → 메모리 변수 대신 .timers/ 파일

5. Develop 브랜치 전략

feature → develop → main 단계별 검증 (Git Flow)

6. 프로젝트 자동 감지

git remote URL 기반 자동 식별 → DX 개선

7. 실패 메모리 3단계

기록(JSONL) → 컨텍스트 주입 → CLAUDE.md 영구 규칙 승격 (3회 반복 시)

8. 외부 데이터 격리

<external_data> 래퍼로 프롬프트 인젝션 방어
